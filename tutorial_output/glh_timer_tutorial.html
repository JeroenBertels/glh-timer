This tutorial walks through the main admin workflow in GLH Timer using the sample race GLH-Spring-Tri-2026.

  • Admin login used for the screenshots: admin / admin
  • Demo organiser accounts included in the data set: racelead and assistant
  • Race statuses reflect the seeded dates around March 20, 2026

1. Log In

Open the login page and sign in with an admin account or an organiser account linked to the race you want to manage.

2. Races Dashboard

The home screen lists all races and their status. Admin users also see shortcuts to race and organiser management.

3. Manage Races

Use the race management screen to add races, edit existing entries, or import and export the race list as CSV.

4. Manage Organisers

Admin users can create organiser accounts and grant access to one or more races from the organiser management page.

5. Race Overview

Each race overview page links to its race parts, participant administration, and QR code export for check-in and timing.

6. Manage Race Parts

Race parts define the sequence used for results and automated wave starts. The Overall part is generated automatically.

7. Manage Participants

Add participants manually or through CSV import. QR codes can be downloaded per participant for fast timing capture.

8. Review Results

Results pages show rankings per part and overall. Filters let you narrow the table by group or sex.

9. Submit Start Times

Use the start submission form for manual starts. You can enter bib numbers, groups, or ranges in a single action.

10. Automated Wave Starts

Wave starts compute offsets from previous part results. This is useful when the next leg should start in ranking order.

11. Show Timer

The timer screen can be projected on-site. It tracks elapsed time from the selected start event and supports full-screen display.

12. Submit End Times

End times can be entered directly, scanned through QR codes, or captured first and assigned to bibs afterward from the pending queue.

13. Submit Durations

Duration entry is useful when you have elapsed times already measured and want to add them without separate start and end events.

14. Manage Timing Events

The timing event ledger lets organisers review, edit, delete, and bulk import recorded timing data for a race part.
